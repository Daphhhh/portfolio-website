Working Banking System

As part of our Object-Oriented Programming (OOP) course, my professor grouped us and assigned a project to develop a fully functional banking system. This was my first hands-on experience building a complete system using OOP principles.

Our system included features like:

  • Account Creation and Deletion
  • Deposits, Withdrawals, and Fund Transfers
  • Check Balance, Request Loans, and Change PIN
  • Role-Based Access Control (Manager, Tellers, Customer)

We applied core OOP concepts such as encapsulation, inheritance, and polymorphism, and we used MySQL as our database to store and manage data. This project strengthened my understanding of system design, teamwork, and the practical use of OOP in real-world applications.

Working CPU Scheduling Calculator

As part of our OS requirement, we were tasked to create a simple Python application with a graphical interface that simulates four CPU scheduling algorithms: FCFS, SRTF, Priority, and Round Robin. Users can enter process details and see the scheduling results both as average times and a graphical Gantt chart.

Our system included features like:

  • Easy-to-use Tkinter GUI for process input and algorithm selection.
  • Supports FCFS, SRTF, Priority, and Round Robin (with time quantum).
  • Displays Average Waiting Time and Turnaround Time.
  • Shows a clear Gantt chart with process execution timelines and idle periods.
  • Basic input validation and error handling.
